## РОЗДІЛ 3. РЕАЛІЗАЦІЯ СИСТЕМИ УПРАВЛЕННЯ ЗАДАЧАМИ
### 3.1. Серверна частина (Backend)
#### 3.1.1. Налаштування середовища (Node.js, TypeScript, Express)
- Налаштування npm‑скриптів, TypeScript компіляція та запуск серверу
- Конфігурація Prisma, схеми бази даних та міграції PostgreSQL
- Реалізація контролерів, сервісів і маршрутів (/api/auth, /api/groups, /api/tasks, /api/users)
- JWT‑автентифікація, роль‑базований контроль доступу (RBAC)
- Socket‑комунікація (socket.io) для реального часу
- Валідація запитів за допомогою Zod, глобальна обробка помилок
### 3.2. Клієнтська частина (Frontend)
#### 3.2.1. Налаштування Vite, React 19, TypeScript, Tailwind CSS
- Створення компонентної архітектури (Header, KanbanBoard, GroupDashboard, TaskModal, UpdateTaskModal та ін.)
- Управління станом через Zustand і React Query
- Використання REST‑API та WebSocket для CRUD задач, груп та користувачів
- Темізація (dark/light), UI‑ефекти (градієнти, тіні)
- Фільтрація, пошук та швидке додавання задач
- Реалізація ролей у UI (адміністратор, менеджер, користувач)
### 3.3. Бізнес‑логіка
- Робочі процеси задач (статус, пріоритет, залежності)
- Тайм‑трекинг та автоматизація (правила If‑Then)
- Реалізація подій у реальному часі через socket.io
### 3.4. Тестування
- Юніт‑тести бекенду (Jest, ts-node)
- Інтеграційні тести API (SuperTest)
- UI‑тести фронтенду (Cypress або Playwright)
- Тестування socket.io подій

## РОЗДІЛ 4. РОЗГОРТАННЯ ТА ЗАБЕЗПЕЧЕННЯ БЕЗПЕКИ
### 4.1. Підготовка інфраструктури та розгортання
#### 4.1.1. CI/CD у GitHub Actions
- Запуск тестів, збірка Vite та створення артефактів
- Деплой бекенду (Node/Express) та фронтенду (Vite) на Vercel або Netlify
- Використання secrets для environment variables та JWT‑ключів
- Автоматичний preview‑деплой для pull‑request‑ів
#### 4.1.2. Docker‑композ (опціонально)
- Dockerfile для API, веб‑серверу та PostgreSQL
- docker‑compose.yml для локального розгортання та тестування в контейнерах
### 4.2. Захист мережі та сервера
- HTTPS (Let’s Encrypt) та налаштування CORS
- Rate limiting, захист від brute‑force (bcrypt, короткоживучі JWT)
- Захист від SQL‑Injection та XSS через Prisma, Zod та CSP‑заголовки
- RBAC‑контроль доступу на бекенді та фронтенді
### 4.3. Тестування безпеки
- Статичний аналіз коду (ESLint, TypeScript)
- Сканування вразливостей OWASP ZAP для API та UI
- Перевірка прав доступу та ролей
- Тестування socket.io підключень без автентифікації
### 4.4. Моніторинг та логування
- Логи сервера (console, winston) та клієнта
- Health‑check ендпоінт /health
- Моніторинг подій socket.io
**Висновки до розділу 4**
